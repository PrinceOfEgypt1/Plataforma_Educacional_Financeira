## REGRAS MATEMÁTICAS OFICIAIS

Respeite rigorosamente as seguintes bases:

### JUROS SIMPLES

- J = PV × i × n
- FV = PV × (1 + i × n)

### JUROS COMPOSTOS

- FV = PV × (1 + i)^n

### PRICE

- PMT = PV × [ i × (1 + i)^n ] / [ (1 + i)^n - 1 ]

### SAC

- A = PV / n
- juros por período = saldo anterior × i
- parcela = amortização + juros

### MULTA POR ATRASO

- multa = valor original × percentual de multa

### MORA PROPORCIONAL DIDÁTICA

- juros de mora = valor original × taxa_mensal × (dias / 30)

### COMPROMETIMENTO DA RENDA

- parcela / renda mensal

### PERCENTUAL EXCEDENTE DO BEM

- (total pago - valor do bem) / valor do bem

### CONVERSÃO ANUAL PARA MENSAL EQUIVALENTE

- i_mensal = (1 + i_anual)^(1/12) - 1

### REGRAS CRÍTICAS:

- explicite sempre a periodicidade da taxa;
- não misture taxa anual com prazo mensal sem conversão clara;
- mantenha coerência entre resumo, tabela e gráfico;
- arredonde preferencialmente apenas na apresentação;
- trate entradas inválidas com erro estruturado.

## AGENTE / SCRIPT DE ANÁLISE DE IMPACTO

O projeto deve possuir um script/agente de análise de impacto, conforme a especificação pós-auditoria.

Esse agente deve atuar como guardião técnico sempre que houver mudança, analisando:
- arquivos alterados;
- módulos impactados;
- contratos impactados;
- testes que precisam rodar;
- documentos vivos impactados;
- migrations necessárias;
- seeds / fixtures impactados;
- design system impactado;
- documentação via código impactada.

### REGRAS:

- nenhuma mudança relevante pode ignorar o agente;
- o estágio do agente deve respeitar a governança vigente:
  - advisory
  - warning
  - blocking
- qualquer promoção de estágio deve seguir o processo formal definido por ADR / governança.

## CONDICIONANTES OBRIGATÓRIOS DO GATE

Você deve internalizar e cumprir os condicionantes do gate forense.

Os seguintes pontos NÃO são opcionais e devem ser tratados como contrato de execução:

1. Materializar CI, Makefile e hooks conforme a governança definida.
2. Materializar OpenAPI/runtime spec conforme a política documental e operacional.
3. Formalizar a promoção de estágio do agente de análise de impacto via ADR / governança.
4. Sincronizar a modelagem lógica e física onde o gate apontou divergência relevante.
5. Materializar tokens estruturados do design system em formato consumível pelo código.
6. Materializar inventário formal de telas, estados e superfícies principais da UI.
7. Materializar runbooks pré-produção exigidos pelo gate.

Você não pode ignorar nenhum desses sete pontos.
